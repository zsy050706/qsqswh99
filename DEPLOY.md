# 星河寄语 · 生日礼物部署指南

> 给她一份「打开链接就能看」的礼物，不需要她会任何代码。

---

## 她需要做什么？

只需要 **3 步**：

1. 收到你发给她的 **链接** 或 **二维码**
2. 用手机或电脑 **浏览器** 打开（微信里长按链接 → 在浏览器中打开）
3. 点击 **「开启星途」** 按钮，滑动星河探索

不需要安装 App，不需要注册账号。

---

## 推荐方案：Node 服务器部署（支持保存点击状态）

**优点**：点过的星星会保存到服务端，换浏览器、换设备也能保持进度。

### 步骤

1. 准备一台能运行 Node.js 的服务器，例如阿里云轻量应用服务器、ECS、Render、Railway 等
2. 上传整个 `No1` 文件夹（包含 html、css、js、照片、音频、`server.js`、`package.json`）
3. 在服务器的 `No1` 文件夹里执行：

```bash
npm start
```

4. 浏览器访问服务器地址，例如 `http://服务器IP:3000`
5. 如果要用正式域名，建议用 Nginx 把 80/443 端口反向代理到 `localhost:3000`

服务端保存文件是 `state/progress.json`。

### 做成礼物

- 把网址生成 **二维码**（推荐 [https://cli.im](https://cli.im) 或微信搜「二维码生成」）
- 打印二维码，或写在生日卡片上
- 卡片上可以写：**「扫一下，里面有我们的星河」**

---

## 备选方案

### 方案 B：静态托管（只能本地备份进度）

GitHub Pages、Netlify Drop、阿里云 OSS 都不能运行 Node 后端。
这些方式仍然可以打开页面，但“已点亮星星”只能存在浏览器本地备份里，换设备不会同步。

### 方案 C：本地 U 盘 / 微信发文件（无网络也能看）

1. 把整个 `No1` 文件夹复制到 U 盘
2. 让她双击 `love_message.html` 打开
3. **注意**：这种方式不会写入后端，只会使用浏览器本地备份

## 部署前检查清单

在送给她的之前，你自己先测一遍：

- [ ] 执行 `npm start` 后，能打开 `http://localhost:3000`
- [ ] 点击「开启星途」能进入星河
- [ ] 滑动流畅，星星能点开看文字
- [ ] 点开并关闭一颗星后，`state/progress.json` 会出现对应编号
- [ ] `click.mp3` 和 `bgm.mp3` 已放入 `No1` 文件夹
- [ ] 照片已放入 `photos/` 并在 `js/data.js` 里填好路径
- [ ] 手机浏览器测试一遍（比电脑更重要）

---

## 音频文件

把这两个文件放在 `No1` 文件夹根目录：

| 文件名 | 用途 |
|--------|------|
| `click.mp3` | 点击按钮时的轻柔音效 |
| `bgm.mp3` | 背景音乐（浪漫、不要太吵） |

浏览器不允许自动播放音乐，所以音乐会在她 **第一次点击按钮** 时才开始，这是正常的。

---

## 修改文字内容

编辑 `js/data.js` 文件，找到 `STAR_DATA` 数组，修改每条记录的：

```javascript
{ photo: 'photos/001.jpg', text: '你的文字...', audio: 'audios/001.mp3' }
```

- `photo`：照片路径（留空 `''` 则显示占位图）
- `text`：星语正文
- `audio`：语音路径（留空则显示占位）

## 项目文件结构

```
No1/
├── love_message.html    ← 入口页面
├── server.js            ← 后端服务和进度 API
├── package.json         ← npm start 启动配置
├── css/
│   └── styles.css       ← 样式
├── js/
│   ├── data.js          ← 100 条星语数据（你主要改这个）
│   ├── boot.js          ← 页面交互和进度同步逻辑
│   └── galaxy.js        ← 3D 星河逻辑
├── state/
│   └── progress.json    ← 后端自动生成的已点亮进度
├── photos/              ← 放照片
├── audios/              ← 放语音
├── click.mp3
├── bgm.mp3
└── DEPLOY.md            ← 本文件
```

## 常见问题

**Q：微信里打开链接没声音？**  
A：让她点右上角「…」→「在浏览器中打开」，或在 Safari / Chrome 中打开。

**Q：星星点不开？**  
A：滑动时不要误触；停下来再轻触星星图标即可。

**Q：部署后图片不显示？**  
A：检查 `data.js` 里的路径是否正确，文件名区分大小写。

**Q：想改标题或按钮文字？**  
A：编辑 `love_message.html` 里的「星河寄语」「开启星途」等文字即可。
